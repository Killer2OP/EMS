# VidyutSeva EMS - Backend Documentation

This document provides a comprehensive overview of the VidyutSeva Electricity Management System backend, its architecture, available endpoints, and instructions for running and testing the application locally.

## 1. Project Overview

The backend is a robust RESTful API built with **Spring Boot 3.4.3** and **Java 17**. It handles authentication, user management, consumer records, monthly billing, payments, and customer complaints.

### Key Technologies
- **Framework:** Spring Boot 3.4.3 (Java 17)
- **Security:** Spring Security 6 with JWT (`jjwt` library)
- **Database:** H2 In-Memory Database
- **ORM:** Spring Data JPA (Hibernate)
- **Build Tool:** Maven wrapper (`mvnw`)
- **Validation:** Jakarta Bean Validation

---

## 2. How to Run the Backend

You can run the backend locally using the included Maven wrapper. No external database installation is required as it uses an in-memory H2 database.

### Prerequisites
- **Java 17** installed and configured in your system `PATH`.
- A terminal (PowerShell, Command Prompt, or Git Bash).

### Running the Application

1. Open a terminal and navigate to the backend directory:
   ```powershell
   cd c:\Users\prath\Desktop\electricity-ms\backend
   ```

2. Execute the Maven wrapper command to run the Spring Boot application:
   ```powershell
   .\mvnw.cmd spring-boot:run
   ```

3. **Success Check:** You should see console logs culminating in a message similar to:
   ```text
   Started VidyutsevaApplication in X.XXX seconds
   ```
   The backend is now accessible at `http://localhost:8080`.

> [!TIP]
> **Stopping the server:** Press `Ctrl + C` in the terminal to stop the application gracefully.

## 4. API Architecture

The application is divided into five core domains:

### A. Authentication & Security
- **Stateless JWT:** Upon successful login, the server issues a JSON Web Token containing the user's ID, role, and consumer ID.
- **RBAC (Role-Based Access Control):** Every request containing a valid `Bearer` token is authorized based on the `role` claim (`CUSTOMER`, `SME`, or `ADMIN`).

### B. Consumers & Users
- **Users (`UserAccount`):** Contains credentials and roles.
- **Consumers (`Consumer`):** Represents the physical electricity connection (Address, Meter Number, Tariff Type).
- Customers map exactly to one Consumer. Admins and SMEs do not have associated Consumers.

### C. Billing & Tariffs
Bills are generated based on tariff types:
- **Domestic:** ₹5.0 per unit
- **Commercial:** ₹8.0 per unit
- **Industrial:** ₹10.0 per unit
- A unique constraint ensures only one bill is generated per billing period (e.g., `2024-03`) for a consumer.

### D. Payments & Invoices
- Supports both **Online** (card) and **Offline** (cash/card via Admin) payments.
- When a payment is successful, the bill status is updated to `PAID`, and an **Invoice** is automatically generated with a unique reference number (e.g., `INV-20250415-2`).

### E. Complaints (Ticketing System)
- Customers can raise complaints (Categories: BILLING, OUTAGE, METER_FAULT, OTHER).
- Admins can log complaints on behalf of walk-in customers.
- Admins assign `OPEN` complaints to specific SMEs.
- SMEs view their assigned complaints and resolve them, setting the status to `CLOSED`.

## 5. Available API Endpoints

*All secured endpoints require the `Authorization: Bearer <token>` header.*

### Authentication (`/api/auth`)
- `POST /register`: Register a new customer.
- `POST /login`: Authenticate and receive a JWT token.

### Consumers (`/api/consumers`)
- `GET /`: Get all consumers (Admin only).
- `GET /{id}`: Get consumer by ID (Admin only).
- `GET /number/{consumerNumber}`: Get consumer by 13-digit meter ID (Admin only).
- `POST /`: Add a new consumer (Admin only).
- `GET /search?query={text}`: Search consumers by name or number (Admin only).

### Bills (`/api/bills`)
- `GET /`: Get all bills.
- `GET /consumer/{consumerId}`: Get all bills for a specific consumer.
- `GET /consumer/{consumerId}/history`: Get paid bills (history) for a consumer.
- `GET /{billId}`: Get bill by ID.
- `POST /`: Generate a new bill (Admin only).
- `DELETE /{billId}`: Delete a bill.

### Payments (`/api/payments`)
- `POST /online`: Process an online credit/debit card payment.
- `POST /offline`: Process a walk-in payment (Admin only).
- `GET /consumer/{consumerId}`: Get payment history for a consumer.

### Complaints (`/api/complaints`)
- `POST /`: Raise a new complaint (Customer).
- `POST /admin-log`: Log a complaint for a walk-in customer (Admin only).
- `GET /`: Get all complaints.
- `GET /active`: Get `OPEN` and `ASSIGNED` complaints (Admin only).
- `GET /{id}`: Get complaint by ID.
- `PUT /{complaintId}/assign`: Assign complaint to an SME (Admin only).
- `GET /sme/assigned`: Get complaints assigned to the logged-in SME (SME only).
- `PUT /{complaintId}/resolve`: Resolve a complaint (SME only).
- `GET /consumer/{consumerId}`: Get all complaints raised by a specific consumer.
- `GET /sme-list`: Get a list of all SME users for dropdown assignment menus (Admin only).
